# Design QA

Check:

- hierarchy and spacing
- alignment and density
- interaction affordances
- empty/loading/error states
- responsive behavior
- text overflow
- contrast and accessibility basics
- whether the interface matches the domain tone
- whether the first screen supports the user's real workflow
- whether copy, density, and hierarchy match the target audience

Use browser evidence before signing off.

If the result looks polished but does not help the target user complete the core job, treat it as a product issue rather than a visual polish issue.
