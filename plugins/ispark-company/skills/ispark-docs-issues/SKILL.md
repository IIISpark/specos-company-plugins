# ISpark Docs And Issues

Use this for durable written artifacts and issue workflows.

## Route

- Documentation and release notes: read `references/documentation.md`.
- PRD and planning docs: read `references/prd.md`.
- Issue breakdown, triage, or GitHub issue resolution: read `references/issues.md`.
- PDF output: read `references/pdf.md`.

## Defaults

- Write company-facing docs, PRDs, issue plans, and reports in Simplified Chinese unless the target repo, publication channel, or user explicitly requires another language.
- Put temporary deltas, audits, plans, and draft decision notes under `working-delta/` when present; use `.tmp/` or `tmp/` for disposable generated files. Do not write artifacts into Superpowers, hidden assistant, or arbitrary home-directory document paths.
- Durable facts belong in scoped docs, not chat memory.
- Temporary deltas should not become central truth until stabilized.
- Issue work needs reproduction, owner, scope, and closure criteria.
- Release notes must be based on actual changed artifacts and verification.
