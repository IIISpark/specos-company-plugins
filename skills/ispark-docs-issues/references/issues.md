# Issues

Issue breakdown:

- one independently grabbable slice per issue
- clear outcome
- affected files or surfaces
- acceptance criteria
- test/verification command
- dependencies and non-goals

GitHub issue resolution:

- reproduce or confirm the report
- ask for missing details only when required
- reference related commits or PRs
- close only when the fix is verified or the project workflow says closure is appropriate

For batches, group independent issues and avoid overlapping file ownership.
